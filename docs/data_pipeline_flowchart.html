eval_analyzer — Data Pipeline

반도체 Fail-Item 평가 엔진 · evaluate(run_input) 6단계 (L0~L6) 오케스트레이션
INPUT report_server → run_input
파일 1회 run 시 호출. report_server → eval_analyzer 한 방향 (import 없음).
meta (product·lot·wafer·process…) raw_df (측정 + HILIM/LOLIM + XPOS/YPOS + BIN + FAILTNO/TNO)
L0 ingestingest()
run_input → run_id + fail_case 들. 마스터 upsert, item 파싱, 측정 시리즈 메모리 첨부.
fail 식별: FAILTNO==TNO item_class = category · value_type · bin case_id (자연키 sha256)
per fail_case
L1 metricscompute()
raw(메모리)에서 요약통계 계산. raw per-DUT 저장 금지.
cpk / cpl / cpu / cp mean · stdev · min · max yield · bimodality
L2 featurescompute()
engine_version 별 파생 feature.
robust 산포 (MAD) spec margin 공간 (edge·center·quadrant·gradient) site_cpk_delta
L3 signaturesevaluate()
rules(yaml) when_metric 평가 → 발화 signature + evidence + bin context.
←
rules/*.yamlthresholds · signatures · bin_taxonomy
(임계값 하드코딩 금지)
L4 statusdecide()
severity 집계 + severity_bias + trump + specificity.
status: MONITOR·MINOR·MAJOR·CRITICAL confidence data_completeness
GATE should_store ?present.should_store()
저장 대상만 통과. 미통과 → continue (drop).
① yield fail (FAILTNO==TNO) ② cpk < cpk_warn(1.33) marginal (bin=PASS)
L5 recommendfind_precedents() · make_comment()
선례 검색(어댑터) + 템플릿/LLM 합성 → comment.
←
eval.db 선례과거 case_signature · outcome
(sql 기본 / rag 교체형)
L6 presentpersist() · to_result()
RunResult dict 직렬화 + persist 시 eval.db 적재 (raw 제외).
→
eval.db 적재fail_case · raw_metrics · features · evaluation · evidence · signature
OUTPUT RunResult → report_server
저장 주인은 eval_analyzer. report_server 는 반환값을 UI 표시에 사용(선택).
run_id · cases[] status · signature · comment confidence · evidence · precedents join 키 = (item_raw, bin)
입력 파이프라인 단계 (L0~L6) 저장 게이트 출력 룰(yaml) eval.db
